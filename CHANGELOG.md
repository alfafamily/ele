# Changelog

Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
версионирование — [SemVer](https://semver.org/lang/ru/).

**Как выпускать релиз:** обновить `VERSION` и раздел в этом файле → закоммитить
→ поставить тег и запушить его:

```sh
git tag v1.0.0
git push origin v1.0.0
```

Пуш тега запускает workflow `.github/workflows/release.yml`, который создаёт
GitHub Release. Обновление инстансов — по `docs/INSTALL.md` («Обновление версии»).

## [1.0.18] — 2026-07-13

### Изменено
- Смена режима хранилища (локальное ↔ S3) запускает фоновый перенос файлов.
  Теперь пока перенос не завершён:
  - переключение режима заблокировано — на бэкенде (409 при попытке сменить во
    время активного переноса) и в UI (радиокнопки недоступны), чтобы новый
    перенос не пересёкся с текущим и не повредил файлы;
  - в разделе Системные → Хранилище показывается статус переноса (сколько
    файлов осталось), он автоматически обновляется; при ошибках — счётчик и
    кнопка «Повторить перенос».

## [1.0.10] — 2026-07-13

### Добавлено
- Настройки → **Системные**: отдельная вкладка для технических настроек. Из
  «Компании» (там остались только реквизиты — лого, название, ИНН, КПП) сюда
  перенесены хранилище, домен и ограничение по IP.
- Блок «Хранилище файлов»: выбор Локальное/S3 радиокнопкой + «Выполнить

- Ограничение по IP: к каждому адресу теперь можно добавить **примечание**
  (пара полей IP + Примечание, строки добавляются/удаляются).
- Кнопки «Выполнить проверку» для **SMTP** (код на почту админа), **Яндекс ID**
  (доступность по реквизитам из .env) и **Яндекс SmartCaptcha** (решение капчи
  с серверной проверкой ключа). Если сервис не задан в .env — вместо кнопки
  примечание «настройки не заданы».
- Лейбл поля домена изменён на «Домен аккаунтов в системе».

### Изменено

  `company.0004` конвертирует существующие записи, поведение ограничения не

### Добавлено
- Саморегистрация теперь заводит связанного Сотрудника: на форме — Фамилия и Имя
  (обязательны), Отдел и Должность (по желанию); данные маппятся в карточку

## [1.0.4] — 2026-07-13

### Добавлено
- Настройки → Компания → «Проверка отправки почты (SMTP)»: письмо с кодом уходит
  на почту текущего администратора, ввод кода подтверждает реальную доставку
  (аналог шага проверки в мастере первичной настройки).
- Поддержка SMTP на порту 465 (implicit SSL): для порта 465 SSL включается
  автоматически, даже без `EMAIL_USE_SSL` в `.env`; явный `EMAIL_USE_SSL`
  перекрывает автоопределение. `install.sh` спрашивает режим шифрования
  (tls/ssl/none) и выводит его из порта.

### Исправлено
- Отправка приглашения «подвешивала» кнопку с лоадером, если SMTP недоступен:
  добавлен `EMAIL_TIMEOUT` (10 с), отправка обёрнута в обработку ошибок —
  теперь быстрый понятный отказ вместо зависания.
- Приглашение больше не создаёт пользователя, если письмо не удалось отправить
  (создание и отправка — в одной транзакции, при ошибке всё откатывается).

### Обновление
- Если SMTP настроен на **порт 465**, дополнительных действий не требуется — SSL
  определится автоматически. Ручной `EMAIL_USE_SSL=true` в `.env` нужен только
  для нестандартного SSL на другом порту.

### Документация
- `docs/INSTALL.md`: обновления кумулятивны — версии можно перепрыгивать (все
  миграции применятся по порядку за один проход); перед обновлением
  рекомендуется бэкап.

## [1.0.2] — 2026-07-13

Упрощение первичного развёртывания на «голом» сервере.

### Установка
- `install.sh`: если Docker Engine / Compose v2 не найдены, скрипт предлагает
  установить Docker автоматически (официальный `get.docker.com`, под root или
  `sudo`) вместо простого выхода с ошибкой.
- `install.sh`: открывает порты 80/443 в локальном файрволе ОС (ufw/firewalld),
  если он активен, — иначе Caddy не проходит ACME-проверку и HTTPS не
  поднимается. Облачный security group провайдера по-прежнему открывается
  вручную (о чём скрипт предупреждает).

## [1.0.1] — 2026-07-13

Хотфикс по итогам первого боевого развёртывания.

### Исправлено
- Backend навсегда оставался `unhealthy` (и не стартовал `cron` с фоновыми
  задачами): внутренний healthcheck ходит на `Host: localhost`, которого не
  было в `DJANGO_ALLOWED_HOSTS`, — Django отвечал 400. `localhost`/`127.0.0.1`
  теперь всегда в списке разрешённых хостов в проде.
- Тормоза интерфейса (скелетоны по несколько секунд): gunicorn запускался с
  одним синхронным воркером и обрабатывал запросы по одному. Число воркеров
  теперь считается от числа ядер (override — `WEB_CONCURRENCY`,
  `GUNICORN_THREADS`).

### Установка
- `install.sh` устойчив к лимиту Docker Hub (`429 Too Many Requests`):
  предлагает `docker login` и повторяет сборку с нарастающей паузой.
- `docs/INSTALL.md` — раздел «Частые проблемы» (429 при пуле образов и
  диагностика TLS/HTTPS: порты 80/443 и A-запись домена).

## [1.0.0] — 2026-07-13

Первый публичный релиз on-premise системы учёта оборудования, лицензий и
пользователей.

### Основное
- Разделы Оборудование, Лицензии, Сотрудники, Пользователи, Настройки, Профиль;
  роли Администратор / Ответственный за учёт / Сотрудник (+ «Наблюдатель»).
- Типы и динамические реквизиты (EAV), дополнительные поля, файловые реквизиты
  (встроенный просмотрщик, удаление файла).
- Аутентификация: логин/пароль, Яндекс ID, регистрация, приглашения (с
  подтверждением при чужом домене), восстановление пароля, защита от подбора.
- Списки с курсорной пагинацией и адаптивной (desktop/mobile) вёрсткой;
  двухстрочные колонки, обрезка длинного текста.
- Карточки объектов: sticky-боковой блок, действия меню «…» на мобильных,
  история изменений с диффами (что/было→стало), маскирование «Номер/ключ».
- Списание/утилизация с переводом в Архив; списание открепляет сотрудника.
- Хранилище файлов Local/S3 с фоновой миграцией; резервное копирование
  (ручное/по расписанию) и восстановление через CLI.
- Развёртывание: docker-compose (Caddy + авто-TLS), установка «одной строкой»
  (`install.sh`), CI (oxlint + backend-тесты).
