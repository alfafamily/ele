## [1.10.1] — 2026-07-20

### Исправлено
- **Смена Типа лицензии больше не оставляет реквизиты прежнего Типа.** После
  перевода лицензии на другой Тип значения реквизитов старого Типа удаляются
  сразу — раньше, пока старый Тип ещё не удалён, на карточке мог показываться
  «осиротевший» реквизит (например, второй «Номер/ключ»). То же исправлено и для
  смены Типа оборудования.

### Обновление
- Миграция при обновлении **автоматически подчищает** уже накопившиеся
  «осиротевшие» значения реквизитов (для инстансов, где на 1.10.0 успели сменить
  Тип у объектов). Отдельных действий не требуется.

## [1.10.0] — 2026-07-20

### Добавлено
- **Вид Типа лицензии: программная / аппаратная.** Прежние «базовые» Типы
  «Программная» и «Аппаратная» упразднены как захардкоженные — теперь **вид**
  является свойством любого пользовательского Типа лицензии и выбирается при его
  создании. От вида зависит обязательный **ключевой реквизит**: «Номер/ключ»
  (программная) или «Номер/ID/Serial токена» (аппаратная) — он добавляется к Типу
  автоматически. Свои Типы можно свободно переименовывать, архивировать и удалять.
- **Флаг «Установка SIM/E-SIM» у Типа оборудования.** SIM/E-SIM можно установить
  только в оборудование, у Типа которого включён этот флаг. Флаг задаётся при
  создании Типа и меняется на его карточке; подбор оборудования при установке SIM
  показывает только подходящие типы.
- **Открытая регистрация (Настройки → Системные).** Флаг «Открытая регистрация»
  управляет самостоятельной регистрацией: если выключен — зарегистрироваться
  самостоятельно нельзя (и вход через Яндекс ID не заводит новый аккаунт), только
  по приглашению администратора.

### Изменено
- **У лицензии больше нет собственного Наименования** — она идентифицируется
  своим Типом (в списках и карточке показывается имя Типа). Сменить Тип у
  существующей лицензии можно только на Тип **того же вида**.

### Обновление
- Миграция переводит существующие данные автоматически и **без потерь**: прежним
  базовым Типам проставляется вид (Программная → программный, Аппаратная →
  аппаратный) и снимается блокировка; прежнее наименование каждой лицензии
  переносится в её доп. поле **«Прежнее наименование»**. Пустые (без лицензий)
  базовые Типы удаляются; на новых установках стартовых Типов лицензий нет.
- Типам оборудования, у которых уже была установлена SIM, флаг «Установка
  SIM/E-SIM» проставляется автоматически (обратная совместимость).
- Флаг «Открытая регистрация» после обновления **включён** (прежнее поведение
  сохраняется).

## [1.9.0] — 2026-07-20

### Добавлено
- **Размещение объектов: рабочие места, места хранения, склад.** У Места
  (в разделе «Помещения») появился **Тип места** — *Рабочее место* или *Место
  хранения*. За рабочим местом можно закрепить нескольких сотрудников (блок
  «Рабочие места» виден на карточке сотрудника).
- **Три состояния Оборудования и Инструментов.** Объект может быть закреплён
  *за сотрудником* (мобильно), *на рабочем месте* (стационарно, без конкретного
  сотрудника) или лежать *на складе* (свободный остаток). На карточке
  оборудования — единый блок «Размещение» с кнопкой «Разместить/Переместить»;
  при создании размещение выбирается в форме. Списки оборудования получили
  фильтры «За сотрудником / На рабочем месте / На складе».
- **Свободный остаток и склады.** При откреплении Оборудования от сотрудника или
  места указывается место хранения (склад), куда оно кладётся. Для Инструментов
  свободный остаток можно раскладывать по нескольким складам (приход/списание/
  выдача — по желанию с указанием склада); склад необязателен — часть остатка
  может быть «без склада» (общий свободный остаток). На карточке инструмента
  видна разбивка «Свободно по складам» + строка «Без склада».
- **SIM за оборудованием.** Корпоративную SIM теперь можно установить в
  оборудование (например, в модем для резервного канала) — помимо закрепления за
  сотрудником. Свободная физическая SIM/пропуск/ключ хранится на складе.
- **Выбор размещения при создании.** В формах создания SIM, пропусков и ключей
  размещение выбирается сразу — *за сотрудником* или *на складе* (E-SIM — только
  за сотрудником). При создании инструмента для остатка указывается склад, а
  часть остатка можно сразу выдать сотруднику с указанием количества (в истории
  движений отражается, сколько ушло на склад и сколько сотруднику). Благодаря
  этому создание объекта прямо из карточки сотрудника сразу закрепляет его за
  ним.
- **Аппаратные лицензии на складе.** Свободный физический ключ аппаратной
  лицензии можно положить на склад (при создании и при отвязке от оборудования);
  на карточке видно текущее размещение. Программные лицензии — виртуальны, склад
  им не нужен.

### Изменено
- Открепление физических SIM/пропусков/ключей от сотрудника требует указать склад
  (место хранения), куда уходит объект. **E-SIM виртуальна** — при откреплении и
  создании склад не запрашивается.
- **Инструменты:** склад в операциях необязателен, только пока есть свободный
  остаток «без склада» (актуально сразу после обновления инстанса — существующий
  остаток можно списывать/выдавать без указания склада). Когда весь свободный
  остаток распределён по складам, склад-источник обязателен, и объём операции не
  может превышать остаток без склада.
- Кнопка «Открепить» на карточках SIM/пропуска вынесена под сведения о
  сотруднике (раньше не помещалась справа при длинном ФИО).
- **Инструменты — перемещение между складами.** На карточке в ряду с
  «Оприходовать»/«Списать» добавлена кнопка перемещения свободного остатка с
  одного склада на другой (кнопки движения теперь иконками).
- В модалках движения инструмента «Доступно» показывается для выбранного склада
  (а не суммарно по всем).
- Привязка SIM (к сотруднику/оборудованию) и пропусков/ключей (к сотруднику)
  вынесена в модальное окно — единообразно с оборудованием/лицензиями.

## [1.8.0] — 2026-07-19

### Добавлено
- **Новый раздел «Инструменты» — количественный учёт.** Инструмент — это учётный
  объект, описывающий несколько одинаковых единиц (недорогой/расходный инструмент,
  комплектующие) одной карточкой. У объекта только наименование и произвольные
  дополнительные поля — типов и реквизитов типов нет.
- **Карточка инструмента.** При создании задаётся начальный остаток (можно 0).
  На карточке — блок «Остаток» с метриками «Остаток / Свободно / Закреплено» и
  операциями «Оприходовать» (+N) и «Списать» (−N), а также блок «Закреплено за» с
  закреплением/откреплением части единиц за сотрудниками. Один инструмент может
  быть закреплён сразу за несколькими сотрудниками (каждый — своё количество).
  Все движения по количеству попадают в историю (единица «шт.»), комментарий к
  движению необязателен. «Списать» всю карточку переводит инструмент в архив
  (весь остаток списывается, закрепления снимаются) — с подтверждением.
- **Карточка сотрудника** получила блок «Инструменты» — закреплённые за
  сотрудником инструменты строкой «Наименование · N шт.»; при увольнении единицы
  возвращаются в свободный остаток своих карточек.

### Изменено
- **Оборудование остаётся поэкземплярным** — по одному объекту на физический
  экземпляр (как и раньше). Количественный учёт вынесен в отдельный раздел
  «Инструменты» (см. выше), в самом Оборудовании выбора «вида учёта» нет.

### Исправлено
- Модальные окна больше не закрываются, если начать выделение текста внутри поля
  и увести курсор за границы окна (клик по подложке засчитывается только когда и
  нажатие, и отпускание были на самой подложке).

## [1.7.0] — 2026-07-19

### Добавлено
- **Автонумератор учётных номеров** — на форме создания оборудования, ключа и
  пропуска рядом с полем «Учётный номер» появилась кнопка автогенерации: она
  подставляет следующий номер в формате `ПРЕФИКС-номер` (например `KEY-1`).
  У каждого списка объектов свой независимый счётчик; порядковый номер только
  растёт и не переиспользуется — номер «сгорает» при генерации, даже если объект
  в итоге не сохранён. Ручной ввод номера по-прежнему возможен.
- **Настройки → «Префиксы»** — новый раздел (только Администратор): базовые
  префиксы автонумератора для Оборудования (`EQUIP`), Ключей (`KEY`) и Пропусков
  (`PASS`). Префикс можно переопределить; смена префикса не сбрасывает счётчик.
  Префиксы разных видов объектов должны различаться.

### Изменено
- **Уникальность учётного номера пропусков и ключей** — теперь проверяется
  раздельно в разрезе типа объекта: один и тот же учётный номер можно завести
  и у пропуска, и у ключа (раньше номер был уникален сквозно по обоим типам).

## [1.6.0] — 2026-07-18

### Добавлено
- **Архив выданного в карточке сотрудника** — вкладки **«Выдано»** и **«Архив»**.
  На «Архиве» — таблица ранее выданных объектов (оборудование, SIM, средства
  доступа), которые уже откреплены, с датами закрепления и открепления. При
  возврате с карточки объекта активная вкладка и позиция прокрутки сохраняются.
- **Закрепление оборудования с карточки сотрудника** — кнопка «Добавить» в блоке
  «Закреплённое оборудование»: привязать свободное оборудование или создать новое
  (сразу закрепляется за сотрудником).

### Изменено
- **Создание и редактирование SIM и средств доступа** — теперь полноценные
  страницы (как у оборудования и лицензий), а не модальные окна; единый вид форм
  и переходов.
- **Тип SIM/E-SIM** — показывается текстом во второй строке (перед оператором и
  поставщиком) вместо цветной плашки: в списке «Корпоративной связи», Профиле,
  карточке сотрудника и модалке привязки.
- Кнопки «Добавить» в блоках карточки сотрудника — в заголовке блока справа;
  на мобильном — компактная кнопка-иконка «+».
- **Профиль** — блок «Информация о сотруднике» переименован в «Обо мне»; кнопка
  «Сменить пароль» перенесена в блок «Данные учётной записи»; убраны блок
  «Пароль» и ссылка на связанного сотрудника.
- **Имя и фамилия** на мобильном выводятся на отдельных строках (каждая
  обрезается многоточием по ширине) — в списке «Пользователи» и в заголовке
  карточки сотрудника.
- Плейсхолдеры поиска зданий и помещений — просто «Поиск».

### Исправлено
- Кнопка «На главную» на страницах ошибок (404/500) ведёт в «Профиль» (доступен
  всем ролям), а не в «Оборудование».

## [1.5.0] — 2026-07-17

### Добавлено
- **Флаг «Требуется ключ/пропуск» у мест** — место с флагом можно выбрать как
  объект доступа при создании ключа или пропуска. Ключ теперь может указывать на
  конкретное место (здание, помещение **или** место).
- **Колонка «Закреплено за»** в списках SIM-карт и средств доступа — как в
  Оборудовании и Лицензиях.
- **Набор доступа в истории создания** ключа/пропуска — в записи «Объект создан»
  отражаются учётный номер и выбранные здания/помещения/места.

### Изменено
- **Кнопки создания объектов** — текстовый «+» заменён на иконку.
- **Карточки SIM-карты и средства доступа** — блок «Закреплено за» вынесен в
  правую колонку (как в Оборудовании/Лицензиях).
- **Список SIM-карт** — колонки «Оператор»/«Поставщик» убраны (перенесены
  строкой под номер), добавлено «Закреплено за».
- **Список средств доступа** — колонка «Доступ в» перенесена в «Название»; для
  ключей объект доступа показывается как «здание — помещение», для места —
  «Место (здание — помещение)».
- **Логотип в письмах** — встроен как inline-изображение PNG (`cid`) вместо
  ссылки на SVG, который почтовые клиенты не отображали.

### Удалено
- **Поле «Название» у пропусков** — пропуск идентифицируется учётным номером.

## [1.4.0] — 2026-07-17

### Добавлено
- **Блок «Закреплённое оборудование» в Профиле** — сотрудник видит числящуюся
  за ним технику там же, где SIM-карты и средства доступа.

### Изменено
- **Пересмотрены права ролей:**
  - **«Сотрудник»** (без признака) теперь видит только свой **Профиль**
    (закреплённое оборудование, SIM-карты, средства доступа) — бизнес-разделы
    ему недоступны, вход ведёт на Профиль.
  - Признак **«Наблюдатель»** расширен: **просмотр всех разделов** (Оборудование,
    Лицензии, Корпоративная связь, Средства доступа, Помещения, Сотрудники),
    кроме «Настроек» и редактора Типов, строго на просмотр — без каких-либо
    действий. «Номер/ключ» лицензий Наблюдателю не отдаётся и не раскрывается.
- **Руководство пользователя** переработано: актуализирована навигация (нижний
  таб-бар вместо верхней панели), состав и порядок разделов (Профиль — вторым,
  сразу после «Роли и доступ»), таблица ролей; описание регистрации приведено в
  соответствие с логикой (самостоятельная регистрация + приглашение); в разделе
  «Настройки» пояснено про параметры в `.env`; со страниц объектов убраны
  дублирующие упоминания ролей; добавлено предупреждение о недопустимости
  использования ключей/номеров/serial/id лицензий в личных целях; вычитаны
  формулировки.

## [1.3.0] — 2026-07-17

### Добавлено
- **Кнопка «Фильтры» в списках.** Фильтры статуса (Оборудование, Лицензии,
  Корпоративная связь, Средства доступа) убраны под кнопку «Фильтры» рядом с
  поиском — иконка на мобильном, текст на десктопе, бейдж-счётчик при активном
  фильтре. Для Корпоративной связи добавлен фильтр Все / Активные / Неактивные,
  для Средств доступа — Все / Выданные / Неиспользуемые.
- **Поиск в разделе «Помещения»** — по названию и адресу зданий, а также по
  названию/номеру помещений внутри выбранного здания.
- **Пользователи (Настройки):** поиск по имени, фамилии и почте; сортировка по
  умолчанию — сначала активные, затем деактивированные, внутри групп по ФИО
  сотрудника; смена статуса доступа (активация/деактивация) прямо в карточке
  пользователя.

### Изменено
- **Вкладки списков** перенесены в строку поиска (десктоп: вкладки · поиск ·
  фильтры; мобильный: поиск сверху, вкладки под ним). Кнопки «Добавить» и
  «Настроить типы» на мобильном — иконки в строке заголовка.
- **Корпоративная связь и Средства доступа:** вкладка «Неиспользуемые» убрана —
  свободные объекты показываются во вкладке «Активные» и разделяются фильтром.
- **Переименования вкладок:** Оборудование — «Активные» → «Активное»; Сотрудники

  теперь всегда в списке разрешённых хостов в проде.
- Тормоза интерфейса (скелетоны по несколько секунд): gunicorn запускался с
  одним синхронным воркером и обрабатывал запросы по одному. Число воркеров
  теперь считается от числа ядер (override — `WEB_CONCURRENCY`,
  `GUNICORN_THREADS`).

### Установка
- `install.sh` устойчив к лимиту Docker Hub (`429 Too Many Requests`):
  предлагает `docker login` и повторяет сборку с нарастающей паузой.
- `docs/INSTALL.md` — раздел «Частые проблемы» (429 при пуле образов и
  диагностика TLS/HTTPS: порты 80/443 и A-запись домена).

## [1.0.0] — 2026-07-13

Первый публичный релиз on-premise системы учёта оборудования, лицензий и
пользователей.

### Основное
- Разделы Оборудование, Лицензии, Сотрудники, Пользователи, Настройки, Профиль;
  роли Администратор / Ответственный за учёт / Сотрудник (+ «Наблюдатель»).
- Типы и динамические реквизиты (EAV), дополнительные поля, файловые реквизиты
  (встроенный просмотрщик, удаление файла).
- Аутентификация: логин/пароль, Яндекс ID, регистрация, приглашения (с
  подтверждением при чужом домене), восстановление пароля, защита от подбора.
- Списки с курсорной пагинацией и адаптивной (desktop/mobile) вёрсткой;
  двухстрочные колонки, обрезка длинного текста.
- Карточки объектов: sticky-боковой блок, действия меню «…» на мобильных,
  история изменений с диффами (что/было→стало), маскирование «Номер/ключ».
- Списание/утилизация с переводом в Архив; списание открепляет сотрудника.
- Хранилище файлов Local/S3 с фоновой миграцией; резервное копирование
  (ручное/по расписанию) и восстановление через CLI.
- Развёртывание: docker-compose (Caddy + авто-TLS), установка «одной строкой»
  (`install.sh`), CI (oxlint + backend-тесты).
