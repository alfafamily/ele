# ELE

Веб-сервис учёта оборудования, лицензий и сотрудников компании (on-premise, B2B).
Одна установка обслуживает одну организацию.

## Возможности

- Учёт оборудования, лицензий, сотрудников и пользователей с ролями доступа
  (Администратор / Ответственный за учёт / Сотрудник, признак «Наблюдатель»).
- Пользовательские типы и реквизиты объектов, дополнительные поля, файлы
  (со встроенным просмотрщиком).
- Аутентификация по логину/паролю и через Яндекс ID, приглашения пользователей,
  восстановление пароля, защита от подбора.
- История изменений объектов, архив (списание/утилизация).
- Резервное копирование (вручную и по расписанию), восстановление через CLI.
- Хранилище файлов: локальная ФС или S3.

## Установка на сервер

«Одной строкой» (проверит Docker, склонирует репозиторий, интерактивно создаст
`.env` и поднимет стек):

```sh
curl -fsSL https://raw.githubusercontent.com/alfafamily/ele/main/install.sh | bash
```

Требования к серверу, ручная установка, обновление версии и восстановление из
резервной копии — в [`docs/INSTALL.md`](docs/INSTALL.md).

## Локальный запуск

```sh
cp .env.example .env      # заполнить как минимум DJANGO_SECRET_KEY
docker compose up --build
```

- Приложение: http://localhost/
- API health-check: http://localhost/api/health/
- Перехват почты в dev (mailpit): http://localhost:8025/

## Стек

Django + DRF, PostgreSQL, React (Vite) SPA, Caddy (реверс-прокси, авто-TLS),
docker-compose.
