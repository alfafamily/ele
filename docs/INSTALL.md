# Установка ELE на сервер

ELE поставляется как self-hosted решение (docker-compose): одна копия обслуживает
одну компанию. Ниже — минимальные требования и развёртывание «одной строкой».

## Требования к серверу

| Ресурс | Минимум | Рекомендуется |
|---|---|---|
| ОС | Ubuntu 24.04 LTS | Ubuntu 24.04 / 26.04 LTS |
| CPU | 2 vCPU | 2–4 vCPU |
| ОЗУ | 2 ГБ | 4 ГБ (сборка фронтенда при деплое требует ~1–2 ГБ) |
| Диск | 20 ГБ SSD | 40 ГБ SSD + запас под файлы и резервные копии |
| ПО | `git`, `curl`; Docker Engine 24+ и Compose v2 (скрипт установки поставит сам, если их нет) | то же |

Про диск: помимо ОС и Docker-образов (~2–3 ГБ) место расходуют база PostgreSQL,
загруженные файлы (аватары до 2 МБ, файлы реквизитов до 20 МБ) и резервные копии.
При большом объёме файлов вынесите хранилище в S3 (`ELE_STORAGE_MODE=s3`) — тогда
диск сервера почти не растёт.

Сеть: домен с A-записью на IP сервера, открытые порты **80** и **443** — Caddy
сам получит и продлит TLS-сертификат Let's Encrypt. Локальный файрвол ОС
(ufw/firewalld) скрипт установки откроет сам; **облачный security group в панели
провайдера открывается отдельно** — скрипт до него не достанет.

**Локальный режим без домена (HTTP без TLS).** Если домен не нужен (инстанс во
внутренней сети), оставьте «Домен инстанса» пустым — скрипт определит IP-адреса
сервера и настроит доступ по HTTP (порт 80, без TLS). Внешний IP добавляется
только по подтверждению. ⚠️ В этом режиме вход и данные идут **открытым
текстом** — используйте только в доверенной сети; **не** выставляйте так админку
в интернет.

## Установка «одной строкой»

```sh
curl -fsSL https://raw.githubusercontent.com/alfafamily/ele/main/install.sh | bash
```

Скрипт:
1. проверит Docker и Docker Compose; если их нет — **предложит установить Docker
   автоматически** (официальный скрипт `get.docker.com`; нужен root или `sudo`);
2. склонирует репозиторий в `/opt/ele` (или обновит, если уже есть);
3. если `.env` ещё нет — **интерактивно спросит параметры и создаст `.env` сам**
   (домен, почта/SMTP, режим хранилища, при желании — Яндекс Captcha/ID и первый
   администратор); `DJANGO_SECRET_KEY` и пароль PostgreSQL генерируются
   автоматически. Секреты хранятся только в `.env` и не вводятся через интерфейс;
4. откроет порты **80/443** в локальном файрволе (ufw/firewalld), если он активен;
5. соберёт и поднимет прод-стек: `docker compose -f docker-compose.prod.yml up -d --build`.

Каталог установки можно переопределить: `ELE_DIR=/srv/ele bash install.sh`.

### Два способа создать первого администратора и компанию

Команда установки одна и та же; отличается только, как заполнить шаг «Первый
администратор»:

- **Через Setup Wizard (в браузере).** Оставьте email администратора **пустым**
  (Enter). После старта откройте `https://<домен>` — запустится мастер первичной
  настройки (§4.1): он создаст администратора и компанию и проверит интеграции.
  В `.env` учётные данные администратора при этом не пишутся.
- **Через терминал.** Укажите email и пароль администратора на шаге установки —
  администратор создастся автоматически при первом старте. **Пароль после старта
  стирается из `.env`** (он нужен только для разового создания учётки).

Секреты интеграций (SMTP, S3, Яндекс) и **адрес инстанса** в обоих случаях
задаются только в `.env` (адрес нужен ещё до старта — по домену Caddy получает
TLS, а Django проверяет `ALLOWED_HOSTS`). Задать домен → HTTPS; оставить пустым →
локальный режим по IP без TLS (см. выше). Изменить позже — правка `.env`
(`SITE_ADDRESS`, `DJANGO_ALLOWED_HOSTS`, `CSRF_TRUSTED_ORIGINS`, `SITE_URL`) +
`up -d` и, для домена, перенастройка A-записи DNS.

### Куда кладётся `.env`

В корень установки — `/opt/ele/.env` (права `600`). Чтобы поменять параметры
позже: отредактируйте файл и примените:

```sh
cd /opt/ele
docker compose -f docker-compose.prod.yml up -d
```

Переменные читаются один раз при старте контейнера — после правки `.env`
обязателен перезапуск (`up -d`). Полный список переменных с пояснениями —
в [`.env.example`](../.env.example).

## Установка вручную (эквивалент)

```sh
git clone https://github.com/alfafamily/ele.git /opt/ele
cd /opt/ele
cp .env.example .env      # заполнить SITE_ADDRESS, ELE_ADMIN_*, секреты
docker compose -f docker-compose.prod.yml up -d --build
```

## Первый вход

- Если в `.env` заданы `ELE_ADMIN_EMAIL` / `ELE_ADMIN_PASSWORD` — администратор
  создаётся автоматически при первом старте.
- Иначе при первом заходе в браузере откроется **мастер первичной настройки**:
  создание администратора, реквизиты компании, проверка интеграций из `.env`.

## Обновление версии

```sh
cd /opt/ele
git pull --ff-only
docker compose -f docker-compose.prod.yml pull
docker compose -f docker-compose.prod.yml up -d --build
```

Схемные изменения БД применяются миграциями автоматически при старте backend.

Обновления **кумулятивны**: версии можно перепрыгивать (например, с 1.0.2 сразу
на 1.0.9) — при старте backend применит все накопившиеся миграции по порядку за
один проход, поэтапно проходить каждую версию не нужно. Если какой-то релиз
потребует особого порядка обновления, это будет явно указано в его записи в
`CHANGELOG.md`. **Перед обновлением рекомендуется сделать резервную копию** (см.
ниже «Резервное копирование и восстановление») — на случай отката.

## Резервное копирование и восстановление

- Копии создаются из интерфейса (**Настройки → Резервное копирование**) —
  вручную или по расписанию. Файл — JSON со всеми объектами (пароли — хэшами)
  плюс ссылки на файлы.
- **Восстановление — только через CLI** (в интерфейсе не предусмотрено), на
  пустой БД / инстансе в режиме обслуживания:

```sh
docker compose -f docker-compose.prod.yml exec backend \
  python manage.py restore_backup /путь/к/backup.json
```

## Диагностика

```sh
docker compose -f docker-compose.prod.yml ps        # статус сервисов
docker compose -f docker-compose.prod.yml logs -f backend
curl -fsS https://<домен>/api/health/               # health-check API
```

### Частые проблемы

- **`429 Too Many Requests` при скачивании образов.** Лимит анонимных пулов
  Docker Hub (~100 за 6 ч на IP; на IP хостера за NAT легко исчерпан не вами).
  Решение: `docker login` под бесплатным аккаунтом (лимит 200 и привязан к
  учётке) и повторить `up -d --build`; либо подождать ~6 ч. `install.sh`
  предлагает вход и сам повторяет сборку с паузой. Для частых развёртываний —
  registry-mirror в `/etc/docker/daemon.json`.
- **HTTPS не поднимается, работает только HTTP.** Caddy получает сертификат
  Let's Encrypt автоматически при трёх условиях: A-запись домена указывает на
  IP сервера, **порты 80 и 443 открыты снаружи** (чаще всего 443 закрыт в
  firewall облачной панели/`ufw`), заход по домену, а не по IP. Причину видно
  в `docker compose -f docker-compose.prod.yml logs caddy` (строки `acme`).
